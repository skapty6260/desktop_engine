# Прототип проекта
Для начала мне нужно:
1. Сервер для окон (Wayland, XWayland)
2. Рендеринг сцены

Т.к пк сломан и для общего архитектурного плана проекта  я делаю весь проект не монолитным wayland compositor'ом, а модульным проектом, который можно запускать как и просто сервер для работы с окнами и удаленным доступом для линукс машин, так и отдельным рендерером и системой ввода для такого-же удаленного доступа.

## **Начальная структура такая**: 
(Все компоненты не связаны друг с другом напрямую, и могут использоваться на удаленных машинах при помощи IPC, но могут быть включены, отправляя запросы с сервера на рендерер через DBUS или вызывая методы из dynamic libraries .a.so)

Должен быть модуль IPC, но он будет привязан к конкретной машине.
**Для каждого модуля должен присутствовать модуль коммуникации с другими модулями**
Например, в window_server'е у меня wayland сервер для линукса, а в rendering рендер vulkan на винде через glfw окно. Мне и для рендера и для wayland сервера нужны отдельные методы коммуникации с друг другом, дабы wayland отправлял данные об окнах на рендеринг и получал input действия от рендера, передавая их wayland клиенту.

## Проблема 1 (Share IPC)
Придется писать комплексную IPC систему, которая будет затрагивать каждый метод проекта для каждого модуля, дабы на лету менять функционал.

**Реализация**
Делаю систему плагинов, которые будут заменять способ коммуникации. Плагин remote_rendering добавит функционала транслировать рендер на другую удаленную машину, а можно через root_rendering через dbus или header файлы рендера передавать просто методы напрямую в код рендера если он на 1 машине.

# IPC config system
**Для локальной передачи, если все в 1 процессе**
ipc:
  transport: "shared_memory"
  endpoint: "desktop_engine_compositor"

**Для удаленной передачи между разными девайсами**
ipc:
  transport: "network_socket"
  endpoint: "192.168.1.100:8080"

**DBus linux-to-linux**
ipc:
  transport: "dbus"
  endpoint: "org.desktop_engine.Compositor"
